Documentação app walkthrough

Passo 1 : Hello World !

1) Crie uma nova pasta webapp que conterá todas as fontes do aplicativo que criaremos ao longo deste tutorial.

2) Portanto, nos referimos a essa pasta como “pasta do aplicativo” . Agora crie um novo arquivo HTML raiz chamado index.html na pasta do seu aplicativo.

3) Um documento HTML consiste basicamente em duas seções: Header & Body.

4) A parte principal será usada pelo navegador para processar o documento. Usando meta tags, podemos influenciar o comportamento do navegador. Nesse caso, diremos ao navegador para usar UTF-8 como o conjunto de caracteres do documento.

Também daremos ao nosso aplicativo um título que será exibido no navegador.

5) Esteja ciente de que nosso título embutido em código pode ser anulado pelo aplicativo, por exemplo, para mostrar um título no idioma do usuário. A parte do corpo descreve o layout da página.

6) Em nosso caso, simplesmente exibimos “Hello World” usando uma tag p.

>> Dica

Normalmente, o conteúdo da pasta do webapp é implantado em um servidor da Web como um pacote de aplicativo.

Ao implantar a própria pasta webapp, o URL para acessar o arquivo index.html contém webapp no ​​caminho.

7) Convenções Nomeie o arquivo HTML raiz do aplicativo index.html e localize-o na pasta webapp.

Passo 2 : Bootstrap

>> Observação

SAPUI5 é uma biblioteca JavaScript que pode ser carregada do mesmo servidor Web em que o aplicativo reside ou de um servidor diferente.

Se SAPUI5 for implantado em outro lugar no servidor ou você deseja usar um servidor diferente, então você precisa ajustar os caminhos correspondentes no bootstrap (aqui: src = "/ resources / sap-ui-core.js").

Neste tutorial de acordo com suas próprias necessidades. Você pode usar esta referência para a versão estável mais recente do SAPUI5 para o tutorial ou para fins de teste, mas nunca use isso para uso produtivo. Em um aplicativo real, você sempre deve especificar uma versão SAPUI5 explicitamente.

Para obter mais informações sobre o CDN, consulte Variante para inicialização da rede de distribuição de conteúdo.

webapp/index.html

<!DOCTYPE html>
          <html>
          <head>
            <meta charset="utf-8">
            <title>SAPUI5 Walkthrough</title>
          	<script
              id="sap-ui-bootstrap"
              src="https://openui5.hana.ondemand.com/resources/sap-ui-core.js"
              data-sap-ui-theme="sap_belize"
              data-sap-ui-libs="sap.m"
              data-sap-ui-compatVersion="edge"
              data-sap-ui-async="true"
              data-sap-ui-onInit="module:sap/ui/demo/walkthrough/index"
              data-sap-ui-resourceroots='{
                "sap.ui.demo.walkthrough": "./"
              }'>
          
            </script>
          </head>
          <body>
          <div>Hello World</div>
          
          </body>
          </html>

Nesta etapa, carregamos a estrutura SAPUI5 de nosso servidor da web local e inicializamos os módulos principais com as seguintes opções de configuração:

  1. O atributo src da tag <script> informa ao navegador onde encontrar a biblioteca principal SAPUI5 - ele inicializa o tempo de execução SAPUI5 e carrega recursos adicionais, como as bibliotecas especificadas no atributo data-sap-ui-libs .

  2. Os controles SAPUI5 suportam temas diferentes, escolhemos sap_belize como nosso tema padrão.

  3. Especificamos a biblioteca de IU necessária sap.m contendo os controles de IU necessários para este tutorial.

  4. Para fazer uso da funcionalidade mais recente do SAPUI5, definimos a versão de compatibilidade como borda.

  5. Configuramos o processo de “bootstrapping” para ser executado de forma assíncrona.

  6. Isso significa que os recursos SAPUI5 podem ser carregados simultaneamente em segundo plano por motivos de desempenho.

  7. Definimos o módulo a ser carregado inicialmente de forma declarativa. Com isso, evitamos código JavaScript executável diretamente no arquivo HTML. Isso torna seu aplicativo mais seguro.


    Criaremos o script ao qual este se refere mais adiante nesta etapa. Dizemos ao SAPUI5 core que os recursos no namespace sap.ui.demo.walkthrough estão localizados na mesma pasta que index.html . Isso é, por exemplo, necessário para aplicativos executados no SAP Fiori launchpad.

Fazemos isso para evitar o código executável diretamente no arquivo HTML por motivos de segurança.

Este script será chamado pelo index.html . Definimos lá como um módulo de forma declarativa.
